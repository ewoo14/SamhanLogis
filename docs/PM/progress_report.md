## v1.2 → v2.0 변경사항 요약

1. **팀 구조 전면 개편**: 기능별 대팀(26명) → 마이크로서비스 단위 소팀(54명, 12 서비스팀 + PM + UI/UX + DevOps)
2. **팀 구성 원칙**: 서비스마다 TL 1 + BE 1 + FE 1 + QA 1 (4명)
3. **필수 문서 3종**: 기획서, 중간 보고서, 개발 보고서 — 모든 작업에 의무 적용
4. **QA 스크린샷 필수**: QA 보고서에 스크린샷 1장 이상 미첨부 시 미제출 처리
5. **QA 교차 검증**: 연관 서비스 QA 간 교차 검증, FAIL 시 develop 머지 차단
6. **토론 기록 의무화**: 참여자별 발언 상세 기록 + 명확한 결론 필수
7. **데이터 표준**: BaseEntity 7개 감사 필드 (createdAt/By, modifiedAt/By, deletedAt/By, isDeleted)
8. **Soft Delete 전용**: 물리 삭제 금지, isDeleted 플래그 + 자동 필터링
9. **브랜치 전략 개편**: 서비스팀별 독립 브랜치 (team/auth, team/slip 등)
10. **머지 체크리스트 강화**: 필수 문서 제출, QA 보고서, BaseEntity 준수 검토 항목 추가
